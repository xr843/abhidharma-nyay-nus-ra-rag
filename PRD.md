## 一、项目概述

### 1.1 项目背景

《阿毘達磨順正理論》是印度有部论师众贤所造，由玄奘法师翻译，共80卷。此论是对世亲《俱舍论》的系统回应与批判，是研究部派佛教思想的重要文献。

为方便佛学研究者和爱好者学习研究这部重要论典，本项目旨在开发一个基于AI技术的智能资料库，提供语意搜索、智能问答等功能。

### 1.2 项目目标

- 提供基于自然语言的智能问答功能
- 支持多部相关论典的综合检索
- 引用原文段落，确保学术可靠性
- 降低俱舍学研究的门槛

### 1.3 参考资料

- [《瑜伽师地论》资料库](https://ybh.dila.edu.tw/) - 法鼓文理学院
- [CBETA语意搜索](https://cbetaonline.dila.edu.tw/cbss/) - 中华电子佛典协会

#### 3.1.3 功能要点

| 功能项 | 说明 | 优先级 |
|-------|------|--------|
| 自然语言理解 | 理解用户以中文提出的佛学问题 | P0 |
| 语意检索 | 基于向量相似度检索相关段落 | P0 |
| 综合回答生成 | AI生成结构化的综合回答 | P0 |
| 原文引用 | 附带相关原文段落，标明出处 | P0 |
| 相关问题推荐 | 根据问题上下文推荐3-5个相关问题 | P1 |

| 历史记录 | 保存用户的问答历史 | P1 |

使用不同颜色/图标区分四部论典：

| 论典 | 图标 | 颜色建议 |
|-----|------|---------|

### 3.3 相关问题推荐

系统根据当前问题，自动推荐3-5个相关问题供用户继续探索。

**推荐逻辑**：
1. 基于当前问题的关键概念
2. 基于检索到的相关段落
3. 基于佛学知识图谱（如有）

### 3.4 多轮对话

支持用户在同一话题下进行追问，系统保持上下文理解。

**示例**：
```
用户: 顺正理论如何反驳经部的种子说？
AI: [回答...]

用户: 那有部自己是怎么解释业力相续的？
AI: [基于上下文理解，继续回答...]

用户: 普光在论记中是怎么解释这个问题的？
AI: [继续回答...]
```

## 七、API设计

### 7.1 问答接口

**POST** `/api/chat`

请求体:
```json
{
  "question": "顺正理论如何反驳经部的种子说？",
  "session_id": "uuid-string",  // 可选，用于多轮对话
  "history": [                   // 可选，对话历史
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ]
}
```

响应体:
```json
{
  "answer": "众贤在《顺正理论》中对经部种子说的批判主要集中在...",
  "citations": [
    {
      "text_id": "T1562",
      "text_title": "阿毘達磨順正理論",
      "volume": 15,
      "chapter": "辯差別品第二",
      "content": "若谓经部种子者，彼执色心各有种子...",
      "relevance_score": 0.92
    }
  ],
  "related_questions": [
    "有部如何解释业力相续？",
    "俱舍论中世亲对种子说的态度是什么？",
    "普光与法宝在种子说问题上有何不同见解？"
  ],
  "session_id": "uuid-string"
}
```

### 7.2 获取原文上下文

**GET** `/api/context`

参数:
- `text_id`: CBETA编号
- `volume`: 卷号
- `position`: 字符位置
- `range`: 上下文范围(字符数)

响应体:
```json
{
  "text_id": "T1562",
  "volume": 15,
  "chapter": "辯差別品第二",
  "content": "...[上下文内容]...",
  "highlight_start": 100,
  "highlight_end": 200
}
```

### 7.3 获取历史记录

**GET** `/api/history`

参数:
- `session_id`: 会话ID (可选)
- `limit`: 返回数量 (默认20)
- `offset`: 偏移量 (默认0)

响应体:
```json
{
  "sessions": [
    {
      "session_id": "uuid-string",
      "first_question": "顺正理论如何反驳经部的种子说？",
      "message_count": 5,
      "created_at": "2024-12-11T10:30:00Z",
      "updated_at": "2024-12-11T11:00:00Z"
    }
  ],
  "total": 15
}
```
