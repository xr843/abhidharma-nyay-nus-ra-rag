# 优化功能测试指南

## 已实施的优化

### ✅ 优化 1：强化"忠实原文"的 Prompt
**位置**：`backend/app/services/ai_service.py`
**改进**：
- 增加4个负面示例，明确禁止的行为
- 强化"不得改写原文"、"不得推测"、"不得混淆论典"的约束
- 禁用模糊表述词汇（"可能"、"也许"等）

**预期效果**：
- 答案更忠实于原文
- 减少 LLM "脑补"和"推测"
- 当资料不足时，明确说明而非臆造

### ✅ 优化 2：高频问题缓存机制
**位置**：`backend/app/services/cache_service.py`, `backend/app/services/chat_service.py`
**功能**：
- 自动缓存问答结果（30天有效期）
- 相同问题瞬间返回（<1秒 vs 20-50秒）
- LFU 缓存淘汰策略（最少使用的先清理）
- 最大缓存 1000 个问题

**预期效果**：
- 基础问题（"什么是五蕴？"）命中率 10-20%
- 命中时响应速度：**20-50秒 → <1秒**

### ✅ 优化 3：异步预检索
**位置**：`backend/app/services/prefetch_service.py`, `backend/app/services/chat_service.py`
**功能**：
- 用户阅读答案时，后台预取第一个相关问题的完整答案
- 预取的答案自动缓存
- 用户点击相关问题时，瞬间返回

**预期效果**：
- 点击相关问题：**20-50秒 → <1秒**
- 不阻塞主流程
- 提升学习体验（连续深入学习时非常流畅）

### 第四步：测试"忠实原文"优化

1. **提问一个概念性问题**：
   ```
   什么是随眠？
   ```

2. **观察答案质量**：
   - ✅ 是否保持了原文的文言表述（如「隨眠是遠無差別因」）？

3. **提问一个复杂问题**（测试是否混淆论典）：

   ```
   《俱舍论》和《顺正理论》对随眠的理解有何差异？
   ```

   观察：
   - ✅ 是否明确区分了不同论典的观点？
   - ✅ 是否标注了观点来源（如「[依据一]《俱舍论》认为...」）？
